**H Is the Problem**

If we want AI to help people live better lives, we need to get one thing straight:
What exactly are we optimizing?
In machine learning, everything starts with a loss function.
If you define it poorly, the model will optimize the wrong thing — perfectly.

Same goes for happiness.
If we want AI systems to support human well-being, we need a serious definition of function H — the function that represents happiness.
It can’t just mean “more positive emotions.”

A realistic model must include: 
• emotional well-being (feeling good),
• meaning and value alignment,
• real relationships,
• autonomy and agency,
• long-term stability (because people adapt).

And there are hard problems: 
• no universal scale to compare happiness across people,
• hedonic adaptation,
• fairness vs total optimization,
• the “wireheading” risk — maximizing pleasure while destroying meaning.

AI will optimize exactly what we encode.
So before we scale intelligent systems to “improve humanity,”
we need to define — carefully — what improvement actually means.
Otherwise, we’re just tuning gradients in the dark.
